# SPEC-LOTTO-029: 회차별 상세 보기

## HISTORY

- 2026-05-29 v0.1.0: 최초 작성

## 메타데이터

| 항목 | 값 |
|------|-----|
| 도메인 | 조회 / 표시 |
| 영향 범위 | API (`GET /api/draws/{drw_no}`), 웹 페이지 (`/draw/{drw_no}`, `/collect`) |
| 의존 SPEC | SPEC-LOTTO-001(수집), SPEC-LOTTO-022(당첨금 크롤링) |

## 배경 및 목적

현재 `/collect` 페이지는 수집된 회차 목록을 보여주지만, 특정 회차의 상세
정보(당첨 번호, 보너스, 1등 당첨금, 당첨자 수)를 한 화면에서 집중적으로
확인할 수 있는 화면이 없다. 사용자는 회차 단위로 결과를 살펴보고 싶어도
목록 테이블의 한 행만 볼 수 있을 뿐이다.

본 SPEC은 특정 회차의 상세 정보를 한눈에 보여주는 API와 HTML 페이지를
추가한다. 당첨 번호와 보너스를 색상 볼로 시각화하고, 1등 당첨금/당첨자 수를
표시하며, 이전/다음 회차로 이동할 수 있는 네비게이션을 제공한다.
즐겨찾기 번호가 설정되어 있으면 해당 회차 번호와 대조하여 일치 여부를 보여준다.

이 기능은 순수 조회/표시 전용이며, 회차 데이터를 편집하거나 별도의 통계를
계산하지 않는다.

## 요구사항 (EARS)

### REQ-DRW-001: 회차 상세 API (Event-Driven)

WHEN 클라이언트가 `GET /api/draws/{drw_no}`를 요청하면,
시스템은 해당 회차의 상세 정보를 JSON으로 반환해야 한다(SHALL):

- `drwNo`: 회차 번호 (정수)
- `drwNoDate`: 추첨일 (문자열)
- `numbers`: 당첨 번호 6개 리스트
- `bonus`: 보너스 번호 (정수)
- `prize1Amount`: 1등 당첨금 (정수 또는 null)
- `prize1Winners`: 1등 당첨자 수 (정수 또는 null)

### REQ-DRW-002: 없는 회차 API 응답 (Unwanted Behavior)

IF 요청한 `drw_no`에 해당하는 회차 데이터가 존재하지 않으면,
THEN 시스템은 HTTP 404 응답을 반환해야 한다(SHALL).

### REQ-DRW-003: 회차 상세 HTML 페이지 (Event-Driven)

WHEN 사용자가 `GET /draw/{drw_no}`에 접속하면,
시스템은 해당 회차의 상세 HTML 페이지를 표시해야 한다(SHALL):

- 당첨 번호 6개 + 보너스 번호를 색상 볼(시각화)로 표시
- 1등 당첨금과 1등 당첨자 수 표시 (데이터 있을 때)
- 이전 회차(`/draw/{drw_no-1}`)와 다음 회차(`/draw/{drw_no+1}`)로
  이동하는 네비게이션 링크 제공

### REQ-DRW-004: 네비게이션 경계 처리 (State-Driven)

WHILE 첫 회차(최소 회차) 또는 마지막 회차(최대 회차)를 표시하는 상태에서,
시스템은 존재하지 않는 방향의 네비게이션 링크를 비활성화하거나
표시하지 않아야 한다(SHALL).

- 첫 회차에서는 "이전 회차" 링크 비활성화/숨김
- 마지막 회차에서는 "다음 회차" 링크 비활성화/숨김

### REQ-DRW-005: 즐겨찾기 번호 대조 (State-Driven)

WHILE 즐겨찾기 번호가 설정되어 있는 상태에서,
시스템은 해당 회차의 당첨 번호와 즐겨찾기 번호를 대조하여
일치하는 번호를 시각적으로 강조 표시해야 한다(SHALL).

IF 즐겨찾기 번호가 설정되어 있지 않으면,
THEN 대조 표시를 생략해야 한다(SHALL).

### REQ-DRW-006: 수집 목록에서 상세 이동 링크 (Event-Driven)

WHEN 사용자가 `/collect` 페이지의 회차 목록에서 특정 회차 번호를 클릭하면,
시스템은 해당 회차의 상세 페이지(`/draw/{drw_no}`)로 이동시켜야 한다(SHALL).

### REQ-DRW-007: 없는 회차 페이지 처리 (Unwanted Behavior)

IF 사용자가 존재하지 않는 회차의 `/draw/{drw_no}`에 접근하면,
THEN 시스템은 오류(예외)가 아닌 사용자 친화적인 404 HTML 페이지를
표시해야 한다(SHALL).

## Exclusions (What NOT to Build)

- 회차 데이터 편집/수정 기능 없음 (조회 전용)
- 회차별 통계 계산 없음 (단순 표시만, 분석은 기존 `/analyze`가 담당)
- 2~5등 당첨금/당첨자 정보 없음 (1등 정보만 표시)
- 회차 간 비교 기능 없음
- 회차 검색/필터 기능 없음 (단일 회차 직접 접근만)
- 인증/권한 기능 없음

## 기술적 제약

- Python 3.11, FastAPI + Jinja2
- CSV/JSON 파일 저장소 사용 (DB 미사용)
- 기존 수집 데이터 로딩 헬퍼(`lotto/web/data.py`) 재사용
- 1등 당첨금 데이터는 SPEC-LOTTO-022의 크롤링 결과 활용 (없으면 null)
- 즐겨찾기 번호는 기존 설정(`/settings`) 데이터 활용
- 테스트 커버리지 85% 이상 유지
