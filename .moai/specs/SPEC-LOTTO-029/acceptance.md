# SPEC-LOTTO-029 인수 기준: 회차별 상세 보기

Given/When/Then 형식의 인수 시나리오. 모든 시나리오는 자동화 테스트로 검증 가능해야 한다.

## 시나리오 1: 회차 상세 API 정상 조회 (REQ-DRW-001)

- **Given** 1100회차 데이터가 당첨 번호 `[1, 7, 15, 23, 38, 44]`,
  보너스 `9`, 1등 당첨금 `2,500,000,000`, 당첨자 `12명`으로 존재하고
- **When** `GET /api/draws/1100`을 요청하면
- **Then** HTTP 200과 함께 다음을 만족하는 JSON이 반환된다:
  - `drwNo` == 1100
  - `drwNoDate`는 추첨일 문자열
  - `numbers` == `[1, 7, 15, 23, 38, 44]`
  - `bonus` == 9
  - `prize1Amount` == 2500000000
  - `prize1Winners` == 12

## 시나리오 2: 당첨금 정보 없는 회차 (REQ-DRW-001, 엣지 케이스)

- **Given** 1101회차는 당첨 번호는 있으나 1등 당첨금 데이터가 없고
- **When** `GET /api/draws/1101`을 요청하면
- **Then** HTTP 200과 함께 `prize1Amount` == null, `prize1Winners` == null이며
  `numbers`, `bonus`는 정상 반환된다

## 시나리오 3: 없는 회차 API 404 (REQ-DRW-002, 엣지 케이스)

- **Given** 9999회차 데이터가 존재하지 않고
- **When** `GET /api/draws/9999`를 요청하면
- **Then** HTTP 404가 반환된다

## 시나리오 4: 회차 상세 페이지 렌더링 (REQ-DRW-003)

- **Given** 1100회차 데이터가 존재하고
- **When** 사용자가 `/draw/1100`에 접속하면
- **Then** HTTP 200과 함께 상세 페이지가 렌더링된다:
  - 당첨 번호 6개와 보너스 번호가 색상 볼로 표시됨
  - 1등 당첨금과 당첨자 수가 표시됨
  - 이전 회차(`/draw/1099`)와 다음 회차(`/draw/1101`) 링크가 존재함

## 시나리오 5: 첫 회차 네비게이션 경계 (REQ-DRW-004, 엣지 케이스)

- **Given** 최소 회차가 1회차이고
- **When** 사용자가 `/draw/1`에 접속하면
- **Then** "이전 회차" 링크가 비활성화되거나 표시되지 않으며,
  "다음 회차" 링크는 정상 표시된다

## 시나리오 6: 마지막 회차 네비게이션 경계 (REQ-DRW-004, 엣지 케이스)

- **Given** 최대 회차가 1100회차이고
- **When** 사용자가 `/draw/1100`에 접속하면
- **Then** "다음 회차" 링크가 비활성화되거나 표시되지 않으며,
  "이전 회차" 링크는 정상 표시된다

## 시나리오 7: 즐겨찾기 번호 대조 강조 (REQ-DRW-005)

- **Given** 즐겨찾기 번호가 `[1, 15, 44]`로 설정되어 있고
  1100회차 당첨 번호가 `[1, 7, 15, 23, 38, 44]`인 상태에서
- **When** 사용자가 `/draw/1100`에 접속하면
- **Then** 당첨 번호 중 1, 15, 44가 즐겨찾기 일치로 시각적으로 강조된다

## 시나리오 8: 즐겨찾기 미설정 시 대조 생략 (REQ-DRW-005, 엣지 케이스)

- **Given** 즐겨찾기 번호가 설정되어 있지 않은 상태에서
- **When** 사용자가 `/draw/1100`에 접속하면
- **Then** 페이지는 정상 렌더링되며 즐겨찾기 대조 강조/섹션은 표시되지 않는다

## 시나리오 9: 수집 목록에서 상세 이동 링크 (REQ-DRW-006)

- **Given** 사용자가 `/collect` 페이지에 접속해 있고
- **When** 회차 목록이 렌더링되면
- **Then** 각 회차 번호에 `/draw/{drw_no}`로 이동하는 링크가 존재한다

## 시나리오 10: 없는 회차 페이지 404 HTML (REQ-DRW-007, 엣지 케이스)

- **Given** 9999회차 데이터가 존재하지 않고
- **When** 사용자가 `/draw/9999`에 접속하면
- **Then** 서버 예외(500)가 아닌 사용자 친화적인 404 HTML 페이지가 표시된다

## 시나리오 11: 비정상 회차 번호 입력 (REQ-DRW-007, 엣지 케이스)

- **Given** API/페이지가 실행 중이고
- **When** `/draw/0` 또는 `/draw/-5`처럼 유효 범위를 벗어난 회차에 접근하면
- **Then** 404 HTML 페이지(또는 적절한 검증 응답)가 반환되고 서버 오류가 발생하지 않는다
