# Design Direction — 로또 통계 웹 대시보드

SPEC: SPEC-WEB-001
Date: 2026-05-20
Technology: FastAPI + Jinja2 + Chart.js

---

## Intent Statement

이 대시보드는 로또 번호를 감으로 고르는 도구가 아니라, **통계 데이터를 읽는 도구**다.
사용자는 당첨 번호의 패턴을 보고 싶어하지만, 동시에 "이건 그냥 참고용"이라는 걸 안다.
그 심리적 긴장을 존중하는 디자인이어야 한다.

구체적으로: Bloomberg 터미널이나 네이버 증권처럼 데이터가 주인공이고,
UI는 데이터를 방해하지 않는 조용한 틀이 되어야 한다.

---

## 1. 색상 팔레트

도박 사이트의 빨강/금색을 피하고, 재무 데이터 분석 도구에서 차용한다.

| 역할 | 이름 | Hex | 용도 |
|------|------|-----|------|
| 배경 | Surface | `#F8F9FA` | 페이지 배경 |
| 카드 배경 | White | `#FFFFFF` | 콘텐츠 카드 |
| 테두리 | Border | `#E2E8F0` | 구분선, 카드 테두리 |
| 기본 텍스트 | Ink | `#1A202C` | 제목, 본문 |
| 보조 텍스트 | Muted | `#718096` | 레이블, 설명 |
| 주요 강조 | Slate Blue | `#4A5568` | 활성 탭, 버튼, 헤더 |
| 데이터 주색 | Data Blue | `#3B82F6` | 차트 주 계열 (빈도 높음) |
| 데이터 보조색 | Data Muted | `#CBD5E0` | 차트 보조 계열 (빈도 낮음) |
| 긍정 표시 | Teal | `#0D9488` | 적중 등급, 성공 지표 |
| 경고/면책 | Amber | `#D97706` | 면책 문구, 주의 배너 |

**색상 사용 원칙:**
- 차트 색상은 단색 그라데이션(Data Blue → Data Muted)만 사용 — 무지개색 금지
- 강조색(Slate Blue)은 탐색 요소에만, 데이터 시각화에 혼입하지 않는다
- 면책 배너는 Amber를 사용하되 눈에 거슬리지 않도록 opacity 조정

## 2. 레이아웃 패턴

**탭 기반 단일 페이지 (`/`)**

```
┌─────────────────────────────────────────────────────┐
│  헤더: 로고(텍스트) + 마지막 수집 일자              │
├─────────────────────────────────────────────────────┤
│  탭: [빈도 분석] [최근 패턴] [추천 번호] [시뮬레이션]│
├─────────────────────────────────────────────────────┤
│  탭 콘텐츠 영역                                     │
│  (스크롤 가능, 카드 컴포넌트로 구성)                │
├─────────────────────────────────────────────────────┤
│  푸터: 면책 문구 + 데이터 출처                      │
└─────────────────────────────────────────────────────┘
```

**선택 이유:**
- CLI 4개 명령(collect/analyze/recommend/simulate)을 자연스럽게 4개 탭으로 매핑
- SPA처럼 동작하지만 서버 렌더링(Jinja2)으로 구현 가능 — JS 탭 전환만 필요
- 사이드바는 45개 번호 목록이나 필터가 복잡해질 때 필요한데, 이 도구는 그 수준이 아님

**그리드 시스템:**
- 최대 너비 `1200px`, 좌우 padding `24px`
- 카드는 CSS Grid 2열 또는 3열 (반응형 불필요 — 데스크탑 전용 도구)
- 카드 내부 padding `24px`, 카드 간격 `16px`

## 3. 차트 유형별 설정

### 빈도 분석 탭 — 수평 막대 차트

```
번호 01 ████████████████ 32회
번호 07 ██████████████   28회
...
번호 45 ████             10회
```

- **유형:** Chart.js `bar` (horizontal, `indexAxis: 'y'`)
- **정렬:** 빈도 내림차순 기본, 번호순 토글 가능
- **색상:** 상위 10개 `#3B82F6`, 나머지 `#CBD5E0`
- **높이:** 고정 `canvas` — 45개 번호 모두 표시 (스크롤 없이)
- **툴팁:** "번호 {n}: {count}회 출현 (전체 {total}회 중)"

### 최근 패턴 탭 — 히트맵 스타일 테이블

번호(1-45) × 최근 N회차를 격자로 표시 — 순수 HTML table + CSS background-color
Chart.js 불필요. 출현 시 `#3B82F6`, 미출현 시 `#F1F5F9`.

### 추천 번호 탭 — 카드 테이블

```
┌────────────────────────────────────┐
│ 전략: 고빈도 우선                   │
│ [07] [13] [27] [34] [40] [42]      │
└────────────────────────────────────┘
```

- Chart.js 사용 안 함 — HTML/CSS 카드
- 번호 배지: 원형, 배경 `#4A5568`, 흰 텍스트, 크기 `40px`
- 전략 레이블은 Muted 색상 소문자로 (과장 금지)

### 시뮬레이션 탭 — 도넛 차트 + 요약 테이블

- **유형:** Chart.js `doughnut`
- **데이터:** 1등/2등/3등/4등/5등/미당첨 비율
- 미당첨이 압도적으로 크므로 center label에 "미당첨 {n}%" 표시
- 요약 테이블(HTML)에서 실제 수치 제공
- 면책 문구 카드를 차트 바로 아래에 배치 (숨기지 않음)

## 4. 컴포넌트 라이브러리 권고

**Tailwind CSS CDN (권장)**

```html
<script src="https://cdn.tailwindcss.com"></script>
```

**선택 이유:**
- 빌드 환경 없음 — CDN 한 줄로 모든 유틸리티 클래스 사용
- Jinja2 템플릿에서 `class="..."` 직접 작성, 별도 CSS 파일 최소화
- Bootstrap은 컴포넌트 의존성이 커서 커스터마이징 시 충돌 발생 가능
- 순수 CSS는 그리드/간격 체계를 직접 관리해야 해 유지보수 부담

**추가 의존성:**
```html
<!-- 차트 -->
<script src="https://cdn.jsdelivr.net/npm/chart.js@4"></script>
<!-- 폰트 (한국어 지원) -->
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+KR:wght@400;500;600&display=swap" rel="stylesheet">
```

**빌드 도구 없이 사용 가능한 것만** — Node.js, npm, webpack 불가.

## 5. 타이포그래피

| 용도 | 폰트 | 크기 | 굵기 |
|------|------|------|------|
| 페이지 제목 | Noto Sans KR | 20px | 600 |
| 카드 제목 | Noto Sans KR | 16px | 600 |
| 본문 / 레이블 | Noto Sans KR | 14px | 400 |
| 보조 설명 | Noto Sans KR | 12px | 400 |
| 번호 배지 | Noto Sans KR | 15px | 500 |
| 차트 레이블 | system-ui (fallback) | 12px | 400 |

**한국어 + 숫자 혼용 원칙:**
- 번호(1-45)는 항상 두 자리 표시: `01`, `07`, `45`
- 퍼센트, 횟수 등 통계 수치는 굵게 처리해 텍스트와 구분
- 줄바꿈 시 번호 배지가 잘리지 않도록 `white-space: nowrap`

## 6. 피해야 할 기본값

| 피할 것 | 이유 |
|---------|------|
| 복권 빨강 (`#FF0000`) / 금색 (`#FFD700`) | 도박 사이트 연상 |
| Chart.js rainbow palette (기본 색상) | 데이터 계층 없이 시각적 혼란 |
| 깜빡임, 페이드 인 애니메이션 | 데이터 신뢰도를 낮춤, 분석 도구에 부적합 |
| "행운의 번호!" 같은 카피 | 과장 없이 통계 도구임을 명시 |
| 반응형 모바일 레이아웃 | 이 도구는 데스크탑 분석 용도 |
| 아이콘 과용 | 텍스트 레이블이 더 명확 |
| Chart.js 3D 효과 또는 그림자 | 데이터 왜곡, 가독성 저하 |

## 7. 한국어 UI 텍스트 가이드라인

**탭 레이블:**
- "빈도 분석" (Frequency Analysis)
- "최근 패턴" (Recent Patterns)
- "추천 번호" (Recommendations)
- "시뮬레이션" (Simulation)

**차트/표 레이블 원칙:**
- 단위 명시: "출현 횟수", "백분율(%)", "시뮬레이션 횟수"
- 날짜/회차 형식: "제{n}회" 또는 "{YYYY}-{MM}-{DD}"
- 등급 표시: "1등", "2등", "낙첨" (과장 없이)

**면책 문구 (항상 표시):**
```
이 통계는 과거 데이터 기반이며 미래 당첨을 보장하지 않습니다.
로또는 확률 게임입니다.
```

**빈 상태 메시지:**
- 데이터 없음: "데이터를 먼저 수집하세요. (CLI: `python main.py collect`)"
- 로딩: "데이터를 불러오는 중..."

## 8. 시그니처 디자인 요소

**번호 빈도 그라데이션 배지**

번호 분석 결과를 표시할 때, 각 번호(01-45)의 배지 색상을 빈도에 따라
단색 그라데이션으로 처리한다.

```
낮은 빈도  →  높은 빈도
#E2E8F0   →  #3B82F6
(연한 회색)   (선명한 파랑)
```

- 모든 번호가 한눈에 보이는 `6×8` 격자 (번호 01-45)에서 진하기만으로 상대적 빈도를 즉각 인지
- 별도 차트 없이도 빈도 분포를 직관적으로 전달하는 이 페이지의 **핵심 시각 언어**
- 구현: Python에서 각 번호의 percentile을 계산하여 Jinja2 템플릿에 `style="opacity: {value}"` 또는 CSS class 전달

이 요소가 없는 페이지는 그냥 테이블이지만, 이 요소가 있으면 "통계 분석 도구"처럼 보인다.

## 9. 구현 힌트 (디자인 관점)

Jinja2 템플릿 구조 권고:
```
templates/
  base.html          # 헤더, 탭 nav, 푸터, CDN links
  tabs/
    frequency.html   # 빈도 분석 탭 콘텐츠
    patterns.html    # 최근 패턴 탭 콘텐츠
    recommend.html   # 추천 번호 탭 콘텐츠
    simulate.html    # 시뮬레이션 탭 콘텐츠
```

탭 전환은 JS로 `hidden` class 토글 — FastAPI 서버 요청 없이 클라이언트 사이드 처리.
초기 렌더링 시 모든 탭 데이터를 한 번에 서버에서 렌더링하므로 API 추가 없음.

Chart.js 전역 기본값 (base.html에 설정):
```javascript
Chart.defaults.font.family = "'Noto Sans KR', system-ui";
Chart.defaults.font.size = 12;
Chart.defaults.color = '#718096';
Chart.defaults.plugins.legend.display = false; // 대부분 범례 불필요
Chart.defaults.animation = false; // 애니메이션 비활성화
```
