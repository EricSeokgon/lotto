# SPEC-LOTTO-077 인수 기준 (Acceptance Criteria)

## 손계산 픽스처

### 단일 회차

| 입력 | 1자리 개수 | 비고 |
|------|-----------|------|
| [1,2,3,4,5,6] | 6 | 전부 1자리 |
| [10,20,30,40,41,42] | 0 | 1자리 없음 |
| [1,2,3,10,20,30] | 3 | {1,2,3} |
| [7,8,9,10,11,12] | 3 | {7,8,9} |

### 4개 회차 분포 픽스처

- D1: [1,2,3,4,5,6] → 6
- D2: [10,20,30,40,41,42] → 0
- D3: [1,2,3,10,20,30] → 3
- D4: [7,8,9,10,11,12] → 3

계산 결과:
- avg_single_count = (6+0+3+3)/4 = **3.0**
- high_single_pct (count>=3): D1, D3, D4 → 3/4*100 = **75.0**
- most_common_count = **3** (D3, D4 모두 3 → key "3" 빈도 2가 최댓값)

single_distribution:

| key | count | pct |
|-----|-------|-----|
| "0" | 1 | 25.0 |
| "1" | 0 | 0.0 |
| "2" | 0 | 0.0 |
| "3" | 2 | 50.0 |
| "4" | 0 | 0.0 |
| "5" | 0 | 0.0 |
| "6" | 1 | 25.0 |

## 인수 기준 목록

- AC-01: [1,2,3,4,5,6] → "6" count=1, avg=6.0
- AC-02: [10,20,30,40,41,42] → "0" count=1, avg=0.0
- AC-03: [1,2,3,10,20,30] → "3" count=1, avg=3.0
- AC-04: [7,8,9,10,11,12] → "3" count=1 (7,8,9는 1자리)
- AC-05: 9는 1자리, 10은 1자리 아님 (경계 검증)
- AC-06: 보너스 번호는 집계에서 제외된다
- AC-07: 반환 dict는 5개 최상위 키를 모두 포함한다
- AC-08: single_distribution은 정확히 "0".."6" 7개 키를 가진다
- AC-09: 각 분포 항목은 count, pct 두 키를 가진다
- AC-10: 4개 회차 분포 count가 손계산과 일치 ("0"=1, "3"=2, "6"=1)
- AC-11: 모든 버킷 count 합 == total_draws
- AC-12: pct 값은 소수 2자리로 반올림된다
- AC-13: pct 합은 100.0 (부동소수 오차 내)
- AC-14: avg_single_count = 3.0 (4개 픽스처)
- AC-15: most_common_count = 3 (4개 픽스처)
- AC-16: most_common_count 동률 시 작은 값이 이긴다
- AC-17: high_single_pct = 75.0 (4개 픽스처, count>=3)
- AC-18: high_single_pct는 전부 count<3이면 0.0
- AC-19: 빈 draws → total_draws=0, 모든 값 zero, 7개 키 유지
- AC-20: 단일 회차도 정상 집계
- AC-21: 동일 입력 재호출 시 캐시된 동일 객체 반환
- AC-22: invalidate_cache가 _single_digit_cache를 비운다
- AC-23: GET /api/stats/single_digit → 200 JSON
- AC-24: GET /stats/single-digit → 200 HTML
- AC-25: 데이터 부재 시에도 API/페이지 모두 200
