# SPEC-LOTTO-083 인수 기준 (Acceptance Criteria)

## 빈/예외 입력

- AC-01: `get_odd_run_stats([])` → `total_draws == 0`.
- AC-02: 빈 입력 → `has_odd_run_pct == 0.0`.
- AC-03: 빈 입력 → `most_common_group_count == 0`.
- AC-04: 빈 입력 → `avg_odd_run_count == 0.0`.
- AC-05: 빈 입력 → `odd_run_distribution` 4개 키 모두 count=0, pct=0.0.
- AC-06: `get_odd_run_stats(None)` → 예외 없이 빈 구조 반환.

## 헬퍼 `_count_odd_runs`

- AC-07: `[1,3,5,7,9,11]` → 1 (전부 +2 간격 연속, 1 묶음).
- AC-08: `[1,3,9,11,20,30]` → 2 (`{1,3}`, `{9,11}`).
- AC-09: `[2,4,6,8,10,12]` → 0 (홀수 없음).
- AC-10: `[1,5,9,13,17,21]` → 0 (간격 4, 연속 홀수 아님).
- AC-11: `[1,3,7,9,13,15]` → 3 (`{1,3}`,`{7,9}`,`{13,15}`, 캡 3).
- AC-12: `[1,3,2,4,11,13]` → 2 (홀수=[1,3,11,13] → `{1,3}`,`{11,13}`).
- AC-13: `[3,5,7]` 포함 입력 `[3,5,7,8,10,12]` → 1 (홀수=[3,5,7]).
- AC-14: `[1,3,...]` 최소 길이 2 묶음 1개; 단일 홀수(길이1)는 0.
- AC-15: 입력이 정렬되지 않아도 내부 정렬로 올바르게 집계.

## 응답 구조

- AC-16: 반환 dict 는 5개 최상위 키를 모두 포함한다.
- AC-17: `odd_run_distribution` 은 항상 4개 고정 키만 포함한다.
- AC-18: 각 분포 항목은 count·pct 두 키를 가진다.
- AC-19: 모든 분포 count 합은 total_draws 와 같다.
- AC-20: pct 는 소수 2자리로 반올림된다(33.33 형태).

## 4-회차 손계산 픽스처

D1 `[1,3,5,7,9,11]`=1, D2 `[1,3,9,11,20,30]`=2, D3 `[2,4,6,8,10,12]`=0,
D4 `[1,5,9,13,17,21]`=0.

- AC-21: 분포 count — `'0'`=2, `'1'`=1, `'2'`=1, `'3'`=0.
- AC-22: 분포 pct — `'0'`=50.0, `'1'`=25.0, `'2'`=25.0, `'3'`=0.0.
- AC-23: `has_odd_run_pct == 50.0` (D1·D2 2건/4건).
- AC-24: `avg_odd_run_count == 0.75` ((1+2+0+0)/4).
- AC-25: `most_common_group_count == 0` ('0'이 2회 최빈).
- AC-26: 동률 시 더 작은 키 선택(1묶음·2묶음 각 1회 → 1 선택).

## 캐시

- AC-27: 동일 입력 재호출 시 캐시된 동일 객체 반환(`is`).
- AC-28: `invalidate_cache` 후 새 결과 객체 생성(`is not`, 값은 동일).
- AC-29: `invalidate_cache` 가 `_odd_run_cache` 를 비운다.

## 라우트

- AC-30: GET `/api/stats/odd_run` → 200 + 키 구조.
- AC-31: GET `/api/stats/odd_run` 데이터 부재 시에도 200 (`total_draws==0`).
- AC-32: GET `/stats/odd-run` → 200 (HTML).
- AC-33: GET `/stats/odd-run` 데이터 부재 시에도 200.

## 캡 동작

- AC-34: 4개 묶음이 산출되는 경우에도 `"3"` 키로 저장된다(캡 검증).

## 실데이터 스모크

- AC-35: 실데이터 존재 시 `total_draws>0`, `avg_odd_run_count` 0~3 범위,
  `most_common_group_count in (0,1,2,3)`.
