# SPEC-LOTTO-075 인수 기준

회차별 본번호 6개 중 5의 배수 포함 개수(0~6) 분포 분석에 대한 인수 기준.

## 데이터 계층 (get_mult5_stats)

- **AC-075-001**: `[5,7,15,20,33,44]` → 5의 배수 `{5,15,20}` → count=3.
- **AC-075-002**: `[5,10,15,20,25,30]` → 6개 전부 → count=6 (최댓값).
- **AC-075-003**: `[1,2,4,7,8,11]` → 5의 배수 없음 → count=0 (최솟값).
- **AC-075-004**: `[3,5,7,10,11,13]` → `{5,10}` → count=2.
- **AC-075-005**: 보너스 번호(예: bonus=5)는 5배수 개수 집계에서 제외된다.
- **AC-075-006**: 반환 dict는 `total_draws`, `avg_mult5_count`,
  `most_common_count`, `high_mult5_pct`, `mult5_distribution` 5개 최상위 키 포함.
- **AC-075-007**: `mult5_distribution` 은 항상 `"0"`..`"6"` 7개 키를 포함한다.
- **AC-075-008**: 각 분포 항목은 `count`·`pct` 두 키를 가진다.
- **AC-075-009**: 모든 버킷 count 합은 `total_draws` 와 같다.
- **AC-075-010**: 각 pct 는 소수 2자리 반올림이며 합은 100.0(부동소수 오차 내).

## 파생 지표

- **AC-075-011**: 4-회차 픽스처(개수 3,6,0,2) → `avg_mult5_count == 2.75`.
- **AC-075-012**: 동률 시 더 작은 개수가 `most_common_count` 로 선택된다.
- **AC-075-013**: 동률이 아닐 때는 최빈 개수가 그대로 선택된다.
- **AC-075-014**: 4-회차 픽스처 → `high_mult5_pct == 50.0` (count>=3 인 2건/4건).
- **AC-075-015**: `avg_mult5_count`·`high_mult5_pct`·각 `pct` 는 2자리 반올림.

## 경계 및 캐시

- **AC-075-016**: 빈 draws → 예외 없이 일관된 zero 구조, 7개 키 존재,
  `most_common_count=0`, `avg_mult5_count=0.0`, `high_mult5_pct=0.0`.
- **AC-075-017**: 단일 회차도 정상 집계된다.
- **AC-075-018**: 동일 입력 재호출 시 캐시된 동일 객체(`is`)를 반환한다.
- **AC-075-019**: `invalidate_cache()` 후에는 새 결과 객체를 생성하며
  `_mult5_cache` 가 비워진다.

## 라우트

- **AC-075-020**: `GET /api/stats/mult5` → 200 JSON (키 구조 포함, 빈 데이터도 200);
  `GET /stats/mult5` → 200 HTML ("5배수" 또는 "5의 배수" 포함, 빈 데이터도 200).
