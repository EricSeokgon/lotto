# SPEC-LOTTO-076: 4의 배수 포함 개수 분포 분석

## HISTORY

- 2026-06-12 (v0.1.0): 최초 작성 (Planned). 회차별 본번호 6개(보너스 제외) 중
  **4의 배수(4, 8, 12, ..., 44)에 해당하는 번호의 개수**(0~6)를 산출하고, 전체
  이력에 대해 7개 구간(`"0"`~`"6"`)의 분포를 분석하는 읽기 전용 통계 기능으로 정의.
  SPEC-073(3의 배수)·SPEC-075(5의 배수)의 `data.py` 확장 패턴을 그대로 따른다.

## 개요

각 회차의 당첨번호 6개(보너스 제외)에 대해 각 번호가 **4의 배수인지**(`n % 4 == 0`)
판별한 뒤, 한 회차 안에 4의 배수가 **몇 개** 포함되어 있는지를 센다. 이 "4배수 개수"는
0(4의 배수가 하나도 없음)부터 6(여섯 번호가 모두 4의 배수)까지의 정수 값을 가진다.
전체 이력에 대해 이 개수가 0~6 중 어디에 분포하는지를 분석한다.

### 1~45 범위의 4의 배수

1~45 범위에서 4의 배수는 다음 11개이다:

```
4, 8, 12, 16, 20, 24, 28, 32, 36, 40, 44
```

따라서 한 회차의 본번호 6개가 무작위라면 4배수 개수의 기대값은 약
`6 * 11/45 ≈ 1.47` 부근이며, 대부분의 회차는 0~3개에 분포한다.

## 요구사항 (EARS)

### Ubiquitous Requirements

- **REQ-M4-001**: 시스템은 회차별 본번호 6개(보너스 제외) 중 4의 배수
  (`n % 4 == 0`) 개수를 산출해야 한다.
- **REQ-M4-002**: 시스템은 4배수 개수를 `"0"`~`"6"` 7개 고정 키로 분류한 분포를
  제공해야 하며, 미관측 키는 0으로 채워야 한다(zero-fill).
- **REQ-M4-003**: 시스템은 `total_draws`, `avg_mult4_count`, `most_common_count`,
  `high_mult4_pct`, `mult4_distribution` 5개 최상위 키를 반환해야 한다.

### Event-driven Requirements

- **REQ-M4-010**: WHEN 클라이언트가 `GET /api/stats/mult4`를 호출하면, 시스템은
  분포 통계를 200 JSON으로 반환해야 한다.
- **REQ-M4-011**: WHEN 클라이언트가 `GET /stats/mult4`를 호출하면, 시스템은
  분포 통계를 200 HTML 페이지로 렌더링해야 한다.
- **REQ-M4-012**: WHEN 신규 추첨 데이터가 적재되면, 시스템은 4배수 분포 캐시를
  무효화해야 한다.

### State-driven Requirements

- **REQ-M4-020**: WHILE 분석 대상 draws가 비어 있으면, 시스템은 `total_draws=0`,
  `avg_mult4_count=0.0`, `most_common_count=0`, `high_mult4_pct=0.0`, 7개 키 전부
  0인 일관된 빈 구조를 반환해야 한다.

### Unwanted Behavior

- **REQ-M4-030**: 시스템은 보너스 번호를 4배수 개수 계산에 포함해서는 안 된다.

## 파생 지표 정의

- **avg_mult4_count**: 회차별 4배수 개수의 산술 평균(소수 2자리 반올림).
- **most_common_count**: count 최댓값을 갖는 개수. 동률 시 더 작은 값이 이긴다.
- **high_mult4_pct**: 4배수 개수가 3 이상인 회차 비율(%, 소수 2자리 반올림).

## 범위

- 본 기능은 SPEC-073(3의 배수)·SPEC-074(짝수)·SPEC-075(5의 배수)와 독립적인
  별개 집계이다.
- 읽기 전용 통계로, 미래 출현을 예측하거나 보장하지 않는다.
