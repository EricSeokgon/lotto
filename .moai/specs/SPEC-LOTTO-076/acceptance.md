# SPEC-LOTTO-076 인수 기준 (Acceptance Criteria)

손계산 픽스처 기반 인수 기준. 4의 배수 = {4,8,12,16,20,24,28,32,36,40,44} (11개).

## 4배수 개수 계산 정확성

- **AC-01**: `[4,8,12,16,20,24]` → 4배수 6개 (전부) → count=6.
- **AC-02**: `[1,2,3,5,7,9]` → 4배수 0개 → count=0.
- **AC-03**: `[4,5,6,7,8,9]` → 4배수 {4,8} → count=2.
- **AC-04**: `[10,20,30,40,41,42]` → 4배수 {20,40} → count=2.
- **AC-05**: 보너스 번호는 집계에서 제외된다(bonus=4여도 본번호 0개면 0).

## 응답 구조

- **AC-06**: 반환 dict는 `total_draws`, `avg_mult4_count`, `most_common_count`,
  `high_mult4_pct`, `mult4_distribution` 5개 최상위 키를 포함한다.
- **AC-07**: `mult4_distribution`은 항상 `"0".."6"` 7개 키를 포함한다.
- **AC-08**: 각 분포 항목은 `count`·`pct` 두 키를 가진다.

## 4-회차 픽스처 (D1~D4)

D1 [1,2,3,5,7,9]→0, D2 [4,8,12,16,20,24]→6, D3 [4,5,6,7,8,9]→2,
D4 [10,20,30,40,41,42]→2.

- **AC-09**: 분포 count는 `"0"=1, "2"=2, "6"=1`, 나머지 0.
- **AC-10**: 버킷 count 합 == total_draws == 4.
- **AC-11**: pct는 `"0"=25.0, "2"=50.0, "6"=25.0`, 나머지 0.0.
- **AC-12**: pct 합은 부동소수 오차 내에서 100.0.
- **AC-13**: `avg_mult4_count` == (0+6+2+2)/4 == 2.5.
- **AC-14**: `most_common_count` == 2 (D3,D4에서 2회 → 최빈).
- **AC-15**: `high_mult4_pct` == count>=3 인 D2 1건 / 4건 == 25.0.

## 파생 지표 경계

- **AC-16**: 동률 시(전부 1회) 더 작은 개수가 most_common_count로 선택된다.
- **AC-17**: 모든 회차 count<3 이면 `high_mult4_pct == 0.0`.
- **AC-18**: 수치 필드는 소수 2자리 반올림.

## 경계 및 캐시

- **AC-19**: 빈 draws → 예외 없이 일관된 zero 구조(7개 키 전부 0).
- **AC-20**: 단일 회차도 정상 집계된다.
- **AC-21**: 동일 입력 재호출 시 캐시된 동일 객체를 반환한다.
- **AC-22**: `invalidate_cache()` 후 새 결과 객체를 생성하며 `_mult4_cache`를 비운다.

## 라우트

- **AC-23**: `GET /api/stats/mult4` → 200 JSON + 키 구조.
- **AC-24**: `GET /api/stats/mult4` 은 데이터가 없어도 200 (total_draws=0).
- **AC-25**: `GET /stats/mult4` → 200 HTML.
- **AC-26**: `GET /stats/mult4` 은 데이터가 없어도 200(빈 상태).
