# SPEC-LOTTO-028 인수 기준: 번호 조합 분석기

Given/When/Then 형식의 인수 시나리오. 모든 시나리오는 자동화 테스트로 검증 가능해야 한다.

## 시나리오 1: 정상 조합 분석 (REQ-CMB-001)

- **Given** 충분한 과거 당첨 데이터가 로드되어 있고
- **When** `POST /api/analyze-combination`에 `{"numbers": [3, 7, 14, 22, 35, 42]}`를 전송하면
- **Then** HTTP 200과 함께 다음을 만족하는 JSON이 반환된다:
  - `sum` == 123
  - `odd_count` == 3 (3, 7, 35), `even_count` == 3 (14, 22, 42)
  - `range_distribution` == `{"1-10": 2, "11-20": 1, "21-30": 1, "31-40": 1, "41-45": 1}`
  - `consecutive_count` == 0 (연속 번호 없음)
  - `frequency_score`, `recent_score`, `companion_score`는 0.0 이상의 실수
  - `verdict`는 `"balanced"`, `"hot"`, `"cold"` 중 하나

## 시나리오 2: 연속 번호 카운트 (REQ-CMB-001)

- **Given** 과거 데이터가 로드되어 있고
- **When** `{"numbers": [3, 4, 5, 20, 21, 40]}`를 전송하면
- **Then** `consecutive_count` == 3 이 반환된다
  - (3,4), (4,5), (20,21) → 연속 쌍 3개

## 시나리오 3: 과거 유사 회차 매칭 (REQ-CMB-001)

- **Given** 특정 회차의 당첨 번호가 `[1, 2, 3, 4, 5, 10]`인 데이터가 있고
- **When** `{"numbers": [1, 2, 3, 4, 5, 45]}`를 전송하면 (5개 일치)
- **Then** `historical_match`에 해당 회차가 포함된다
  - 매칭 회차는 최대 5건까지만 반환된다
  - 5개 미만 일치 회차는 포함되지 않는다

## 시나리오 4: 입력 개수 오류 (REQ-CMB-002, 엣지 케이스)

- **Given** API가 실행 중이고
- **When** `{"numbers": [3, 7, 14, 22, 35]}`를 전송하면 (5개)
- **Then** HTTP 422가 반환되고 분석은 수행되지 않는다

## 시나리오 5: 범위 초과 오류 (REQ-CMB-002, 엣지 케이스)

- **Given** API가 실행 중이고
- **When** `{"numbers": [3, 7, 14, 22, 35, 46]}`를 전송하면 (46은 45 초과)
- **Then** HTTP 422가 반환된다
- **And** `{"numbers": [0, 7, 14, 22, 35, 42]}` (0은 1 미만)도 HTTP 422가 반환된다

## 시나리오 6: 중복 번호 오류 (REQ-CMB-002, 엣지 케이스)

- **Given** API가 실행 중이고
- **When** `{"numbers": [7, 7, 14, 22, 35, 42]}`를 전송하면 (7 중복)
- **Then** HTTP 422가 반환된다

## 시나리오 7: 데이터 부재 시 기본값 (REQ-CMB-003, 엣지 케이스)

- **Given** 당첨 데이터가 비어 있는 상태에서
- **When** `{"numbers": [3, 7, 14, 22, 35, 42]}`를 전송하면
- **Then** HTTP 200과 함께:
  - `frequency_score` == 0.0
  - `recent_score` == 0.0
  - `companion_score` == 0.0
  - `historical_match` == `[]`
  - `sum` == 123 (입력 기반 항목은 정상 계산)
  - `verdict` == `"cold"`

## 시나리오 8: hot 판정 (REQ-CMB-004)

- **Given** 특정 번호들이 과거에 매우 자주 출현한 데이터가 있고
- **When** 그 고빈도 번호들로 구성된 조합을 전송하면
- **Then** `frequency_score`가 전체 평균 빈도의 115%를 초과하여
  `verdict` == `"hot"`이 반환된다

## 시나리오 9: numbers 키 누락 (REQ-CMB-002, 엣지 케이스)

- **Given** API가 실행 중이고
- **When** `{"nums": [3, 7, 14, 22, 35, 42]}`처럼 `numbers` 키가 없거나
  body가 리스트가 아닌 요청을 전송하면
- **Then** HTTP 422가 반환된다

## 시나리오 10: 웹 조합 분석 섹션 표시 (REQ-CMB-005)

- **Given** 사용자가 브라우저에서
- **When** `/recommend` 페이지에 접속하면
- **Then** "조합 분석" 섹션과 6개 번호 입력 폼이 렌더링된다

## 시나리오 11: 웹 입력 오류 피드백 (REQ-CMB-006, 엣지 케이스)

- **Given** 사용자가 `/recommend` 페이지의 조합 분석 폼에서
- **When** 6개 미만 또는 중복 번호를 입력하고 분석을 시도하면
- **Then** 분석 요청이 차단되거나 서버 422 응답 시 명확한 오류 메시지가 표시된다
