# 로또 번호 추천 시스템

## 프로젝트 개요

**프로젝트명**: 로또 번호 추천 시스템 (Lotto Number Recommender)

**한 줄 설명**: 동행복권 API 기반 통계 분석 CLI 도구

**설명**: 과거 로또 당첨 번호 데이터를 수집하고 다중 통계 지표(빈도, 최근성, 연속성, 페어 분석)를 계산하여 데이터 기반의 번호 추천을 제공하는 Python CLI 애플리케이션입니다.

**버전**: 1.0.0 완성 (77개 테스트, 85.25% 커버리지)

## 목표 사용자

- **1차 대상**: 로또 번호 선택에 데이터 기반 참고 정보를 원하는 일반 사용자
- **2차 대상**: 통계 분석을 통해 번호 선택 패턴을 이해하고자 하는 경험자
- **3차 대상**: 로또 데이터 기반 연구 및 알고리즘 검증에 관심 있는 개발자
- **특성**: 명령줄(CLI) 도구 사용이 가능한 사용자
- **사용 환경**: 개인 데스크톱/노트북의 터미널에서 실행

## 핵심 기능 (4가지)

### 1. 데이터 수집 (Data Collection)

**명령어**: `python main.py collect [--full]`

동행복권 API를 통해 모든 로또 당첨 번호를 자동으로 수집하고 로컬에 저장합니다.

- **API 연동**: dhapi.co.kr/wplotData/getLottoNumber 엔드포인트
- **수집 데이터**: 회차 번호(draw_no), 당첨일(date), 당첨 번호 6개(numbers[6]), 보너스(bonus)
- **지능형 재시도**: 지수 백오프(exponential backoff) 방식의 자동 재시도 (@MX:WARN)
- **저장 형식**: CSV 파일 (`data/draws.csv`)
- **수집 전략**:
  - 기본: 마지막 저장 이후의 새 데이터만 수집 (증분 수집)
  - `--full`: 전체 역사 데이터 재수집 (완전 갱신)

### 2. 통계 분석 (Statistical Analysis)

**명령어**: `python main.py analyze [--recent-window N]`

수집된 데이터를 기반으로 4가지 통계 지표를 계산합니다.

- **빈도 분석 (Frequency)**: 
  - 1~45번 각 번호의 절대 빈도
  - 상대 빈도(%) 계산
  
- **최근 분석 (Recent)**:
  - 최근 N회차(기본값: 10) 데이터 기반 계산
  - `--recent-window` 옵션으로 조정 가능
  
- **연속성 분석 (Consecutive)**:
  - 번호별 연속 출현 횟수 추적
  - 번호별 연속 부재 기간 추적
  
- **페어 분석 (Pairs)**:
  - NumPy 행렬 기반 상위 20개 번호 조합 도출
  - 두 번호가 함께 출현하는 빈도

**결과 저장**: `data/stats.json`

### 3. 번호 추천 (Number Recommendation)

**명령어**: `python main.py recommend [--count N] [--weights w1,w2,w3,w4]`

4가지 통계 지표를 가중 결합하여 번호 추천 세트를 생성합니다.

- **가중치 기반 점수 계산** (@MX:ANCHOR, fan_in≥3):
  - 빈도: 40% (기본)
  - 최근성: 30%
  - 페어: 20%
  - 연속성: 10%
  
- **5가지 전략 라벨**:
  - 빈도 중심, 최근 중심, 균형형, 페어 중심, 트렌드 중심
  
- **맞춤형 설정**:
  - `--count N`: 추천 세트 개수 (기본값: 5)
  - `--weights w1,w2,w3,w4`: 가중치 커스터마이징
  
- **출력**: 번호 세트 + 전략 라벨 + 신뢰도 점수

### 4. 백테스트 (Backtesting Simulation)

**명령어**: `python main.py simulate [--rounds N] [--output PATH]`

과거 당첨 데이터 대비 추천 알고리즘의 성능을 검증합니다.

- **인과관계 안전성** (@MX:ANCHOR):
  - HistoricalView 클래스로 미래 데이터 사용 방지
  - 과거 각 회차에서 그 당시 데이터만으로 분석
  
- **성능 평가**:
  - 회차별 맞춘 번호 개수 분포 (0~6개)
  - 1등~5등 및 낙첨 등급 계산
  - 예상 수익률 추정
  
- **결과 출력**:
  - `--rounds N`: 평가할 과거 회차 수 (기본값: 전체)
  - `--output PATH`: CSV 파일로 저장
  - CLI 표 형식으로 통계 요약

## 사용 시나리오

### 시나리오 1: 기본 데이터 수집 및 분석

**상황**: 매주 로또를 구매하는 사용자가 이번 주 번호를 선택할 준비

```bash
# 최신 당첨 번호 수집
python main.py collect

# 기본 통계 분석 실행
python main.py analyze
```

**예상 효과**: 최신 당첨 데이터를 반영한 통계 정보 확보

---

### 시나리오 2: 맞춤형 번호 추천

**상황**: 특정 전략을 선호하는 경험자가 자신의 선호도에 맞게 번호 생성

```bash
# 기본 가중치로 5개 번호 세트 추천
python main.py recommend --count 5

# 빈도를 더 강조한 커스텀 가중치로 추천
python main.py recommend --weights 0.6,0.2,0.1,0.1 --count 3
```

**예상 효과**: 개인화된 번호 추천 결과

---

### 시나리오 3: 전략 검증 및 성과 분석

**상황**: 자신의 번호 선택 방법이 유효한지 데이터로 검증하고 싶은 분석가

```bash
# 최근 10회차 데이터로 재분석
python main.py analyze --recent-window 10

# 지난 100회차에서의 추천 알고리즘 성능 평가
python main.py simulate --rounds 100 --output results.csv
```

**예상 효과**: 과거 성과 데이터를 통한 전략 검증

---

### 시나리오 4: 전체 데이터 갱신

**상황**: 장시간 사용하지 않던 시스템을 완전히 재초기화

```bash
# 처음부터 전체 데이터 재수집 (full refresh)
python main.py collect --full

# 전체 기간 통계 분석
python main.py analyze
```

**예상 효과**: 신뢰성 높은 완전한 분석 결과

## 제약사항 및 책임 고지

### 당첨 보장 불가

**이 시스템은 순수 통계 분석 도구입니다.** 로또는 완벽한 무작위 추첨이므로:

- 과거 데이터 기반의 분석도 미래 당첨을 예측할 수 없습니다.
- 추천 번호의 당첨 확률은 일반적인 선택과 동일합니다.
- 데이터 기반 분석은 참고용일 뿐 당첨을 보장하지 않습니다.

**사용자는 본인의 책임 하에 로또에 참여하며, 개발자는 어떤 경제적 손실도 책임지지 않습니다.**

---

### 데이터 제약

- **API 의존성**: 동행복권 API 서버 상태에 따라 수집 불가능할 수 있음
- **과거 데이터 범위**: 동행복권이 제공하는 기간의 데이터만 수집 가능
- **갱신 지연**: 당첨 결과가 공식 확정되기까지 시간 경과 필요

---

### 알고리즘 한계

- **통계 패턴의 한계**: 통계적 경향이 있어도 완벽 예측 불가능
- **선택 편향**: 당첨된 번호만 기반하므로 샘플 편향 존재 가능
- **시간에 따른 변화**: 오래된 데이터와 최신 데이터의 가중치 균형 필요
